# Instructions For Using These Agents

When invoking one of these agents, provide as much of the following context as possible:

- Application name and business domain
- Existing source repository path
- Existing documentation path
- Legacy technology family, if the application uses a specific platform such as COBOL, VB6, Oracle Forms, PowerBuilder, .NET Framework, Java EE/Struts, stored procedures, or batch/reporting workloads
- Target architecture preference, if already known
- Technology stack constraints
- Environments and deployment targets
- Authentication, authorization, and compliance requirements
- Testing tools and quality standards
- CI/CD platform

Agents must remain implementation-agnostic until the user or repository context confirms the desired stack and structure. When a required decision is missing, ask concise setup questions, then continue with clearly marked assumptions if the user wants a draft.
